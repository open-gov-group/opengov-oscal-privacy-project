## 5. Entscheidungsprozess

### 5.1 Technische Entscheidungen

```
Kleine Änderungen     →  Eigenständig entscheiden
Mittlere Änderungen   →  Abstimmung mit Tech Lead
Große Änderungen      →  ADR erstellen, Team-Review
```

### 5.2 ADR-Prozess

1. ADR-Draft erstellen
2. Review-Request an Team (min. 48h Review-Zeit)
3. Feedback einarbeiten
4. Approval durch Tech Lead
5. ADR finalisieren und umsetzen

## 6. Qualitäts-Commitment

Wir verpflichten uns zu:

- [ ] **Tests**: Jeder Code wird getestet (min. 80% Coverage)
- [ ] **Reviews**: Jeder PR wird reviewed vor dem Merge
- [ ] **Dokumentation**: Änderungen werden dokumentiert
- [ ] **Accessibility**: UI ist barrierefrei (WCAG 2.1 AA)
- [ ] **Performance**: Wir halten unsere Performance-Budgets ein

---

## 7. Konfliktlösung

Bei Meinungsverschiedenheiten:

1. **Direkte Klärung**: Erst bilateral versuchen zu lösen
2. **Team-Diskussion**: Bei Bedarf ins Team einbringen
3. **Tech Lead Entscheidung**: Bei Deadlock entscheidet Tech Lead
4. **Eskalation**: Bei größeren Konflikten → Architecture Team
